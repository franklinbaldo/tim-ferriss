**Foco:** Escala mínima de esforço, retorno máximo. Automações e
monetização da infraestrutura existente.

Após analisar o inventário atual de projetos (`franklin-avatar` context),

O foco aqui é o que está funcionando para *indie hackers* agora: APIs pagas,

## Ação 1: Monetizar a API "minha-receita" via RapidAPI

**A Oportunidade:**
Dados de CNPJ e Receita Federal são essenciais para *onboarding* de clientes,
B2B sales e análise de risco no Brasil. O projeto `minha-receita` já faz o
trabalho duro de servir essas informações via API. Hoje, *indie hackers* ganham
dinheiro empacotando APIs existentes em marketplaces como RapidAPI, que cuida
de toda a cobrança, throttling e autenticação.

**O que fazer:**

1. Fazer o deploy do `minha-receita` em uma infraestrutura barata (como
Fly.io ou Render).
2. Criar uma conta de provedor no [RapidAPI](https://rapidapi.com/hub).
3. Cadastrar os endpoints da API com uma documentação básica.
4. Definir preços: Plano gratuito (ex: 100 requests/mês) e um plano pago
(ex: $10/mês para 10.000 requests).

**Esforço necessário:** 1 a 2 dias. O código já existe, é só questão de
infraestrutura e setup no marketplace.

**Potencial de retorno:** $100 - $500/mês passivo, com custo de servidor
próximo a zero.

## Ação 2: Empacotar o "code-review-bot" no GitHub Marketplace

**A Oportunidade:**
O projeto `code-review-bot` (Autonomous Multi-Agent Code Review SaaS) é o
produto perfeito para o modelo B2D (Business to Developer). Em vez de tentar
vender isso via marketing tradicional, a melhor distribuição é o próprio
GitHub Marketplace, onde empresas e desenvolvedores já estão com os cartões
de crédito cadastrados buscando automações de CI/CD.

**O que fazer:**

1. Integrar o bot como um GitHub App oficial se já não estiver.
2. Adicionar o GitHub Marketplace webhook para gerenciar assinaturas
(Billing API).
3. Modelo de negócios: Gratuito para repositórios públicos e $19-$29/mês por
repositório privado ou organização.

**Esforço necessário:** 3 a 5 dias (principalmente integrando o webhook de
pagamento do GitHub/Stripe).

**Potencial de retorno:** Receita recorrente escalável. Se apenas 10
pequenas agências assinarem o plano básico de $29, já são $290/mês.

## Ação 3: "caixaaberta" como um *Data-as-a-Service* (DaaS) ou Newsletter Paga

**A Oportunidade:**
O `caixaaberta` (Pipeline de dados de imóveis da Caixa Econômica Federal)
extrai dados que valem muito dinheiro para investidores imobiliários e
leiloeiros. Em vez de apenas manter o pipeline rodando, empacote o output
(os dados refinados e limpos) como um produto pago. Investidores pagam
alegremente por "leads" mastigados que os poupam tempo de minerar editais.

**O que fazer:**

- **Opção A (Newsletter Automática):** Criar um script que pega os melhores
imóveis semanais do banco de dados e envia via API do Substack para assinantes.
- **Opção B (Acesso ao Banco de Dados/API):** Vender uma licença mensal para
empresas consumirem os dados mastigados via API ou conexão DuckDB/Supabase.

**Esforço necessário:** 2 a 3 dias. Requer apenas um script cron que
consulte o DuckDB e dispare um payload para uma plataforma de email/API externa.

**Potencial de retorno:** Alto ticket. Investidores podem pagar R$ 97 a R$ 297
por mês por inteligência imobiliária exclusiva e atualizada diariamente.
